Current platform pain points
No real certificate verification
Certificates are visually present but have no live database lookup — easily forged, zero employer trust signal.
Zero mobile app ownership
White-labeled app on a 3rd party's infra. If that vendor closes, your students lose access overnight.
No API layer / headless design
Everything is tightly coupled to WordPress. Scaling, integrations, or any rebuild requires touching the entire monolith.
Weak authentication
No OTP, no 2FA. Student accounts and franchise dashboards are protected by passwords alone.
Duplicate domain problem
richgyan.com and richgyan.in run independently, splitting SEO authority and confusing users.
SSL present, MSME registered
Basic security and legal compliance are in place — a solid foundation to build on.
